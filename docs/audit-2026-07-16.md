## P0 — operational: CI verifies less than it appears to

### A1. All 41 MBE sim-integration tests silently skip in CI (medium, but top priority)
`tests/test_{mbe_adapter,mbe_machine,mbe_generate,inverse_mbe,active_mbe,eval_mbe,multitask_mbe}.py`
are gated by `@skipif(not simlink.sim_available())`. `sim_available()` resolves the
sibling `mbe_sim` package via `MBE_SIM_PATH`, defaulting to the hard-coded local
path `c:\Users\Jiaow\Documents\github\MBE sim`. That path exists on this
workstation (so `272 passed` here), but:
- `.github/workflows/ci.yml` does a single `actions/checkout` (seminn only — the
  non-pip-installable sim repo is never fetched), never sets `MBE_SIM_PATH`, and
  runs plain `pytest` with no `-W error` and no strict-skip guard.
- **Reproduced:** `MBE_SIM_PATH=/nonexistent pytest tests/ -q -rs` → **`231 passed,
  41 skipped`**, exit 0. CI is green on both runners with the entire adapter /
  in-silico machine / E2-E3 layer and every sim-gated proof
  (inverse-interval-covers-machine, cost-to-target, exploitation-gap,
  multitool-beats-pooled, D7 honesty) **never executed**.

Consequence: the "272 tests pass" claim in BUILD_STATE/memory is machine-specific;
on a fresh clone or in CI it is "231 pass, 41 skip". A regression in
`evaluate_physics`, the bounds mirror, pathology directions/determinism, or any
§15.2 machinery proof would not be caught by CI.

Fix: add an opt-in strictness switch honored in CI (e.g. `RIG_REQUIRE_MBE_SIM=1`
turns the skip into a hard failure), or check out / point `MBE_SIM_PATH` at the
sim in a CI job and assert a nonzero MBE-test execution count. At minimum emit a
session-level warning so a green badge can't be mistaken for full coverage.

## P1 — genuine defects & spec gaps (medium)

### B1. `RecipeRecord.validate_against` never enforces the simplex sum-to-1 (schema.py:154)
The compositional branch checks only `isinstance(value, Fraction)`; it never
checks that a factor's components sum to 1 or that all declared components are
present. `{'alloy.ga':0.7,'alloy.al':0.7}` (sum 1.4) and `{'alloy.ga':0.7}`
(missing `al`) both pass a validator whose docstring promises "Raises ValueError
on any violation" (implementation-plan §3.1 "simplex, sum-to-1"). Off the normal path
(encoders build compositions feasible-by-construction), but a load-bearing
data-contract gate silently accepts physically-impossible compositions.
Fix: group by `CompositionalVariable`, require all components present, assert
`math.isclose(sum, 1.0, abs_tol=1e-6)`.

### B2. MBE `RecipeRecord` path bypasses the E2 recipe-vs-config split (machine.py:159)
`run()` given a pre-built `RecipeRecord` returns it unfiltered, and validation
runs against `input_schema = RECIPE + MACHINE_CONFIG` vars, so a machine-config
knob (e.g. `gap`) in the recipe passes. It is copied into the emitted
`RunRecord.recipe` but `evaluate_physics` ignores it (drives geometry from
`self.machine_config`). **Reproduced:** a record with `gap=0.025` emits
`recipe.gap=0.025` while `extra.machine_config.gap=0.012` and produces the
*default-gap* physics (`nonuniformity_pct=0.80` vs `0.41` at true gap=0.025) — an
internally inconsistent row that corrupts a forward model trained on it. The dict
path correctly rejects unknown keys; the two input paths disagree. Fix: validate
the recipe against `RECIPE_VARIABLES` only on both paths.

### B3. JSONL ingest: uncaught `KeyError` under `on_error='skip'`; silent-incomplete accept (ingest.py:117)
`_validate_recipe` does bare `recipe.values[f'{var}.{comp}']` indexing. A JSONL
row missing one compositional component raises `KeyError`, which is **not** a
`ValueError`, so the `except ValueError` in `ingest_jsonl` doesn't catch it and
the whole batch aborts — defeating the documented `on_error='skip'` contract. A
row missing a whole continuous input is silently accepted (surfaces much later in
`records_to_arrays`). Both reproduced. Fix: assert all `flat_input_names` present,
raise a catchable `ValueError`.

### B4. Onboarding `input_keys[:-1]` collinearity-drop is order-fragile (spec.py:139 + doc)
`docs/new-process-onboarding.md:112` tells users to fit the GP on
`numeric_input_names[:-1]` to kill sum-to-1 collinearity. `numeric_input_names`
is in **spec-declaration order**, so this is correct only when the single
compositional block is declared last (as in the PECVD example — the doc admits
this). Declare a compositional `blend` **before** a continuous `temp` and `[:-1]`
drops `temp` (a real input) while keeping all three collinear `blend.*` coords →
a **rank-deficient GP design** with a phantom DOF and garbage epistemic/mean, no
error. Fix: add a `gp_input_keys` accessor that drops exactly one component per
compositional variable regardless of order, and point the doc at it.

### B5. Multi-tool model lacks `posterior_cov` → EPIG / batch AL crash on the chamber-matching model (multitask.py:542)
Neither `MultiToolGPForwardModel` nor `ToolBoundForwardModel` implements
`posterior_cov`, but `epig()` and `select_batch()` call it. **Reproduced:**
`epig(model.for_tool('new_chamber'), ...)` → `AttributeError`. `bald()` works
(predict-only), masking the gap. So EPIG/BatchBALD acquisition cannot drive the
ICM model — precisely the §10.4 new-chamber few-shot onboarding this module
exists for. The crash is loud (not silent wrong answer). Fix: implement
`posterior_cov` on the ICM model + forward it from the tool-bound view.

## P1 — load-bearing test-gaps (correct today, regression ships green) (medium)

Each was proven by injecting the described bug and observing the suite stay green.

- **C1. Single-output NLML gradient has no finite-diff test** (test_gp.py) — the
  multitask sibling *is* FD-tested; sign-flip / dropped-noise-grad / ×2 all pass.
- **C2. `posterior_cov` invariant unasserted** (gp.py:334) — dropping the raw-unit
  `ys*ys` factor inflates EPIG ~10× yet the ordering-only EPIG tests pass; the
  batch consumer normalizes it away, so nothing catches a raw-scaling bug.
- **C3. Jackknife+ ±inf tiny-n honesty untested** (conformal.py:215) — the D4
  small-n honesty guarantee; replacing the ±inf guard with an order-statistic
  clamp passes all 9 conformal tests. (Also: every conformal test is m=1.)
- **C4. SBC continuity correction untested** (calibration_gates.py:139) — TARP has
  a small-M regression guard; SBC (all tests M=200) doesn't; removing the
  `(k+U)/(M+1)` jitter passes.
- **C5. Exploitation joint-vs-per-output untested** (exploitation.py:110) — every
  test is m=1 where the two are identical; swapping them mis-reports the §12.1
  joint miss-rate ~2.7× for any real m≥2 campaign, undetected.
- **C6. EPIG magnitude never pinned** (test_active_acquisition.py) — `c**2→c`
  inflates EPIG ~2400× but ordering/sign hold, so both EPIG tests pass.
- **C7. λ blend direction never pinned** (acquisition.py:135) — all blend tests use
  `lam=0.5` (symmetric); swapping EPIG/BALD weights inverts the §9.4
  BALD→EPIG schedule with the full suite green.
- **C8. Acquisition-stall stop path never exercised** (loop.py:222) — OOD BALD
  keeps `max_acq` above `stall_eps`, so the branch is dead in tests and the one
  unreachable-spec test accepts either stop reason; a broken stall predicate ships.
- **C9. M2 assertions carried by the shared seed DoE** (test_m2_comparison.py:70) —
  `sum(bo_e)≥7` is met by 7/10 identical seed-DoE hits *before* any acquisition;
  negating EI (worst-point search) still passes. The BO adaptive core is untested.
- **C10. `expected_improvement` has zero tests** (baselines) — no `test_warm_bo.py`;
  a constant/sign-flipped EI passes the whole suite. The M2 "beats warm-started

- **C11. Multi-output ICM prediction values untested** (test_multitask_gp.py) — all
  fixtures are m=1; a per-output de-standardization mis-alignment on a 2-output

## P2 — low-severity defects, contract nits, and thinner test-gaps

- **D1. Transform `inverse` silently clamps out-of-bounds recipe values**
  (transforms.py:65) — `inverse({'temp':1500})` with bounds [200,1200] returns a
  finite near-boundary `u` with no error; defense-in-depth gap (ingest is the
  authoritative range gate).
- **D2. `forward` returns the exact boundary for |u|≳37** (transforms.py:61) —
  `sigmoid` underflows to 1.0, contradicting the documented open-interval
  bijection; no current consumer computes `log(hi-x)`, so doc/invariant nit.
- **D3. `max_candidates≤0` returns `[]`** (pessimistic.py:512) — a third,
  unspecified result state a caller may misread as INFEASIBLE. Floor to 1 or raise.
- **D4. `_margins` annotation/docstring say 7-tuple, code returns 8**
  (pessimistic.py:346) — invites a 7-value unpack that binds `support` to the σ
  scale. Fix the annotation + docstring.
- **D5. `Δ = delta_frac·(upper−lower)` range factor untested** (pessimistic.py:318)
  — every delta test uses range=1.0; dropping the range factor (600× under-count
  of the §8.5 δ penalty for a real `temp∈[300,900]` var) passes the suite.
- **D6. Pessimistic predicted-outcome interval unasserted** (pessimistic.py:426) —
  only `lo≤hi` is checked; dropping the `s` term or using epistemic σ ships a wrong
  operator-facing worst-case interval undetected.
- **D7. `test_returns_distinct_candidates` vacuous if 1 candidate**
  (test_inverse.py:402) — asserts distinctness but not `len≥2`; a
  regress-to-single-anchor is only caught by a *different* test. Add `len(xs)≥2`.
- **D8. Sobol seed design: loud non-power-of-2 warning + opaque empty-ranges crash**
  (interfaces.py:248) — deterministic, but noisy logs (may trip warnings-as-errors
  CI) and an unhelpful numpy error for `ranges={}`. Add a guard / suppress.
- **D9. `cholesky_with_jitter` escalates to 1e-3·mean-diag but docstring says 1e-4**
  (_gp_common.py:27) — the final un-guarded attempt uses 10× the documented max;
  doc/behavior mismatch, no wrong answer.
- **D10. Compositional unit gate accepts scaled-dimensionless `percent`/`ppm`**
  (spec.py:250) — passes the dimensionless check but ingest ignores the unit and
  reads raw values as fractions (loud `Fraction>1` crash or silent accept of
  already-[0,1] values). Reject anything not literally dimensionless.
- **D11. KM Greenwood `var` field unasserted & unconsumed** (survival.py:107) — a
  Greenwood typo ships green; RMST recomputes its own variance so no blast radius
  until a caller trusts `.var`. Pin it or drop the field.
- **D12. ACI pre-update err-timing not actually pinned** (test_conformal.py:155) —
  the always-miss/always-hit inputs make `err` alpha-invariant, so a post-update
  timing bug would pass. Code is correct; test can't see it.
- **D13. `c_batch` cost accounting untested off-sim** (loop.py:137) — every
  non-gated test uses `c_batch=0`; both shipping adapters feed `c_batch=1000`, so a
  regression corrupts the WP-G/M2 cost metric once real costs are used.
- **D14. M2 never asserts the difference-in-RMST direction** (test_m2_comparison.py:83)
  — with 9/10 seeds identical, the head-to-head signal is one seed and nothing
  guards the headline "RIG cheaper" claim; a RIG regression flips δ silently.
  (The test's docstring already frames it as harness-validity-only — the real gap
  is the absence of any separate performance test.)

## Refuted (adversarial verification did its job)

- **"validate_against never rejects a recipe omitting a declared variable"** —
  literally true, but both concrete adapters' `encode_recipe` raise a clean,
  *tested* `ValueError` naming the missing variable at the natural boundary; the
  validator docstring never promises completeness. Down to a doc nit, not a defect.
- **"difference-in-RMST uses independent variance but the M2 harness feeds paired
  seeds"** — the code faithfully implements the independent Uno-2014 SE **by
  design**: implementation-plan §12.4 assigns the pairing to *separate* tests (paired Wilcoxon
  primary + paired bootstrap), and the M2 harness asserts only well-formedness, so
  nothing is "buried". The residual (paired Wilcoxon/bootstrap not yet built) is a
  missing-feature note, not a bug in `rmst_difference_test`.

## Cross-cutting themes / recommendations

1. **Fix CI first (A1).** Everything else is moot if the integration layer isn't
   actually running anywhere reproducible.
2. **The dominant weakness is unpinned correctness, not wrong math.** ~19 of 33
   findings are "code correct, regression ships green." A short campaign to add
   value-pinning tests (FD gradient, `posterior_cov` invariant, EPIG closed-form,
   λ-direction, ±inf jackknife, SBC small-M, m≥2 exploitation & ICM, EI known-answer,
   range≠1 δ, stall path) would close most of the medium band.
3. **Two systemic test blind spots:** almost every UQ/eval test is **single-output
   (m=1)**, and the **adaptive cores** (BO EI, AL stall) are only exercised where
   the seed DoE already wins. Both hide whole classes of regression.
4. **Nothing here blocks M0/WP-E.** No critical or currently-wrong-answer defect
   was found; the code does what the spec says. These are hardening items.
