# M3 acceptance v2 — honest re-run on the InSilicoMachine (ground-truth scored)

**Gate (implementation-plan §15.4):** *amortized proposal matches per-query quality after
refinement.* Generated by `examples/run_m3_acceptance_v2.py` (deterministic, seeded);
data in `docs/m3-acceptance-v2.json`. Full run 2026-07-22, ~6.4 min wall.

**Verdict: PASS — and, unlike v1, non-tautologically.** On a shared set of 6 targets
where a gap provably exists, ground-truth top-1 hit counts were **cold_heavy 6/6,
cold_light 3/6, d2_light 6/6**. `d2_light` matched the heavy gold standard (6 ≥ 6) and
strictly beat the light cold solver (6 > 3), at **0.1875× the multi-start budget** (9 vs
48 starts) and **0.229× wall-time** (16.6 s vs 72.6 s). The three gap targets were genuine
`cold_light` **INFEASIBLE** abstentions (the single box-centre start never moved); the
amortized generator's proposals warm-started a single 1-restart refinement into a
heavy-quality recipe on all three.

**Read the PASS precisely.** It is a PASS of the §15.4 gate *as written* — *amortized +
one refinement matches heavy quality at light budget* — scored honestly against ground
truth. It is **not** a demonstration that amortization beats equal-budget random search: a
control sweep shows **9 random restarts rescue the same 3/3 gap targets** as d2's 9
(1 + 8 amortized) starts, so d2's edge over `cold_light` here is the restart *budget*
(9 vs 1), not the amortized proposals being better than random ones. On this smooth,
low-dimensional, near-invertible map, amortization's distinctive payoff (a zero-optimization
instant answer; multimodal mode coverage) is not exercised. See Caveats.

This does **not** retroactively strengthen v1. v1's PASS stays on record as the
toy-`tanh`, surrogate-scored, saturating-rule result it was; v2 is a separate, honest
measurement on the real machine.

result). v2 addresses each critique:

| 2 | **No gap:** `cold_light` already hit 4/5, so "amortization fills the gap" rested on n=1. | Targets are **pre-registered via a cheap `cold_light` pre-probe** that deliberately includes 3 targets where the light solver FAILS (boundary pre-images) + 3 HIT controls. ## Setup

- **Machine:** `InSilicoMachine(MBE)`, `PathologyConfig(metrology_noise=True)` (as M2; no
  drift / hidden-state — out of scope for the amortization question). Recipe vars
  `T_heater ∈ [1150,1500] K`, `film_thickness ∈ [2e-7, 5e-6] m`.
- **Ground-truth oracle:** the **noise-free** clean machine physics (deterministic,
  call-order-independent) evaluated at each returned recipe. Never the surrogate.
- **Model outputs (GP + generator):** `T_center`, `bow_cooldown_um` — the two channels
  that carry a genuine, learnable posterior width and jointly pin `(T_heater,
  film_thickness)`. `thickness_grown` (a near-noiseless `film_thickness` identity, SNR
  ~220) was **excluded**: it POISONS SBC calibration (it teaches the flow an overconfident
  `film_thickness` posterior), and the §14.6 gate will not pass with it in (measured — see
  Caveats). This scopes the demonstrated map to **2→2**, not the full 2→5.
- **Amortized generator (§14.3):** zuko NSF deep ensemble, 5 members, 4 transforms,
  hidden (128,128), 400 epochs, `region_hw=(0.25, 2.5)`, trained on **N=1024** seeded-Sobol
  noisy machine runs. (v1 used 220 on a toy and never ran the gate; 1024 is the smallest
  power-of-two in a config sweep that clears the gate with margin — 512 was marginal, SBC
  dim-2 p=0.022. Machine cost stays trivial. This is a gate-prerequisite choice, not
  tuning-to-pass the arm verdict.)
- **§8 solver policy — BINDING:** `kappa = z_epi = 2.0`, `delta_frac = 0.02` (the audited
  binding §8 policy, *not* M2's 1.0/1.0 ablation). All three arms share it; they differ
  only in restart budget / warm-start.
- **Spec boxes:** coupled `(T_center, bow_cooldown_um)`, half-width `0.5 · std(Y_train)`
  per output (= standardized half-width 0.5, on the generator's region-augmentation
  manifold). Half-widths `≈ [39.1 K, 2.65e-5 m]` = **16.7× / 57.7× the metrology σ**
  (`[2.34 K, 4.59e-7 m]`), so a single-shot oracle read is a robust acceptance test.

**Arms** (same spec, same GP model, per target):
`cold_heavy` = 48 cold Sobol restarts (= library default 24·dim at dim=2, the full
budget); `cold_light` = 1 start (box centre) = the cut budget; `d2_light` =
`AmortizedRefiner` — the generator proposes 8 recipes that warm-start the SAME 1-restart
solver (9 starts total).

## §14.6 SBC/TARP gate — BLOCKING, and it PASSED

The amortized generator's `validate()` gate ran **before any arm** (full mode aborts on a
failed gate). Result: **PASS** — `sbc_passed=True` (per-recipe-dim p = [0.9015, 0.394]),
`tarp_passed=True` (`tarp_max_calibration_error = 0.025`), `n_sim=200`, `n_posterior=100`.
The gate certifies the posterior relative to the (noisy) generative model the flow was
trained under (§14.6 Caveat 1: relative to that model, here the in-silico machine — not a
real tool). Calibration attaches to the amortized proposal via this gate, never to the
refined output.

Boxes are the coupled `(T_center [K], bow_cooldown_um [m])` spec. "hit" = the arm's
**top-1** returned recipe, evaluated on the noise-free oracle, lands in the box. Selected
targets = the first 3 `cold_light`-MISS (idx 3,4,5) + first 3 `cold_light`-HIT (idx 0,1,2)
in pool order (4 corners then Sobol).

|---|---|---|---|---|

| idx 5 — ref `(1345 K, 4.40e-6 m)` | MISS | **HIT** (conf 1.000) | **INFEASIBLE** — "predicted mean outside box, −120σ (bow)"; returned box centre | **HIT** (conf 1.000), outcome `(969.5 K, 1.49e-4 m)` |
| idx 0 — ref `(1164 K, 3.92e-7 m)` | HIT | **HIT** (1.000) | **HIT** (1.000) | **HIT** (1.000) |

| idx 2 — ref `(1486 K, 3.92e-7 m)` | HIT | **HIT** (1.000) | **HIT** (0.99975) | **HIT** (1.000) |
| **ground-truth top-1 hits** | | **6 / 6** | **3 / 6** | **6 / 6** |

**Mechanism (why the gap is real).** For the 3 MISS targets the `cold_light` single
box-centre start (`u = 0` → `T_heater = 1325 K, film_thickness = 2.6e-6 m`) never moved:
its INFEASIBLE verdict returns *the box centre itself* as nearest-achievable. These targets
sit at recipe-box corners, whose pre-images are in the reparameterization's saturated
`u`-tail — unreachable by one local descent from the centre. `cold_heavy`'s 48 restarts
find them; `d2_light`'s 8 amortized proposals land near the corner, so a single refinement
locks in a recipe whose *true* outcome matches `cold_heavy`'s. Every d2 top outcome above
is inside its box on the noise-free physics.

## Caveats and what is still open

- **The win is restart BUDGET, not amortized-vs-random starts — the load-bearing caveat.**
  `cold_light = 1` restart is the extreme cut, and the gap it exposes is a **multi-start
  search gap** (one local descent cannot reach a saturated-tail pre-image). A control sweep
  on the 3 MISS targets shows plain **9 random Sobol restarts rescue the same 3/3** (cold at
  `n_restarts` 1/2/3 → 0/3, 5 → 1/3, **9 → 3/3**, 16/48 → 3/3). So on this smooth 2-D map
  d2_light's edge over `cold_light` is the **budget (9 starts vs 1), not the amortized
  proposals being better than random ones** — its 8 amortized warm-starts are *sufficient*
  to close the gap but **not demonstrably superior** to 8 random restarts of equal count.
  The M3 §15.4 gate (*amortized + refine matches heavy quality at light budget*) is met as
  specified; the stronger claim *amortization beats equal-budget random search* is **NOT
  demonstrated here**. Amortization's distinctive payoff — a zero-optimization instant
  answer, and mode coverage on higher-dim / multimodal pre-images — is simply not exercised
  by this low-dim, smooth, near-invertible venue.
- **Confidences saturate at ~1.0** on this smooth, low-noise map (same as v1) — which is
  precisely why the verdict ignores them and scores ground truth. Do not read the conf
  column as evidence; it is reported only for continuity with v1.
- **Scope is 2→2, not the full 2→5.** `thickness_grown` and `slip_max_ratio` were excluded
  because the honest §14.6 gate cannot calibrate with a near-deterministic identity output
  in the set (measured: with all 5 outputs SBC fails hard, p=[0.03, 0.0]; even 4 outputs
  fails). This is a real limitation of the amortized posterior on near-invertible maps, not
  a harness convenience — stated plainly.
- **The gate certifies the full 2-output box** at standardized half-width 1.0; the arms use
  the same 2 outputs at half-width 0.5. Both are on the trained manifold, but the certified
  box is not byte-identical to the served box. (v1's calibration-boundary note applies:
  calibration attaches to the proposal, never the refined output.)
- **In-silico machinery proof, not real-tool evidence** — same standing as M2; the real
  headline stays gated on M0. The gate is certified relative to the in-silico generative
  model, not a real tool (§14.6 Caveat 1).
- **N=1024** (vs v1's 220) was needed to calibrate the gate on the real map; this is a
  prerequisite, chosen from a sweep, not tuned to the arm verdict.

## Reproduce / determinism

```
python examples/run_m3_acceptance_v2.py            # full (writes docs/m3-acceptance-v2.json)
python examples/run_m3_acceptance_v2.py --smoke    # tiny fast config
```

Deterministic (all streams seeded). The JSON's `timing` block is the only
non-deterministic content and is excluded from the `DETERMINISM_DIGEST` the runner prints.
The **smoke config double-runs byte-identical** (timing-stripped) — verified, digest
`6f75c98f…`. The full run's timing-stripped digest is `dff2fddc…` (recorded; the full run
is done once, as briefed).
