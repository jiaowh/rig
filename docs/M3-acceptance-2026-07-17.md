# M3 acceptance ‚ÄĒ amortized proposal matches per-query quality after refinement

**Gate (implementation-plan ¬ß15.4):** *amortized proposal matches per-query quality after refinement.* End-to-end D2 (real GP forward tier + real zuko amortized generator + ¬ß8 pessimistic refinement via `AmortizedRefiner`) on a self-contained coupled in-silico process (plateau-shaped `rate`, so a cold box-centre start is trapped). Deterministic (seed 0). Generated by `examples/run_m3_acceptance.py`.

**Verdict: PASS.** D2-light matched cold-heavy quality on 5/5 feasible targets; D2-light rescued 1/5 targets that the light cold solver missed; refinement budget = 0.281√ó cold-heavy (1 cold start + 8 amortized proposals vs 32 Sobol starts).

- **cold-heavy**: canonical ¬ß8 solver, 32-start cold Sobol multi-start (per-query gold standard).
- **cold-light**: same solver, 1 start (box centre) ‚ÄĒ trapped on the plateau ‚áí the gap amortization fills.
- **D2-light**: amortized generator proposes 8 recipes; the 1-start solver refines them (the D2 `AmortizedRefiner`).
- **raw proposal hit-rate**: fraction of the generator's *unrefined* proposals whose GP-mean already lands in the spec box ‚ÄĒ the amortized quality before ¬ß8 refinement.

**Calibration boundary (D2):** calibration attaches to the amortized proposal (the ¬ß14.6 SBC/TARP gate on the generator) and to the conformally re-validated selected set (the solver's `revalidation_model` + ¬ß13.2 gate) ‚ÄĒ never to the refined output.
